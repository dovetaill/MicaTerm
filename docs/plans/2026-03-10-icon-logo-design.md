# Mica Term Icon And Logo Design

日期: 2026-03-10
执行者: Codex

## 背景

当前项目是基于 Rust + Slint 的 Windows 11 桌面终端工具，已明确采用 `Command Console Fluent` 视觉骨架，窗口材质倾向 `Mica Alt`，主题强调色固定为 `electric-blue`，终端主工作区保持中性。图标与 logo 需要沿用这套约束，避免做成独立于产品风格之外的视觉语言。

用户明确偏好类似现代终端工具的几何图标气质，并提到对 Termius 一类图标有好感。最终目标不是模仿具体现有产品，而是提炼其“干净、几何、小尺寸可识别、技术感强”的优势，形成符合 `Mica Term` 自身定位的标识系统。

## 目标

- 设计一套适合 `Mica Term` 的品牌 logo
- 设计一套适合 Windows 任务栏的小尺寸应用图标
- 保证主 logo 与任务栏图标属于同一视觉系统
- 保证图标在深色优先、浅色兼容的主题策略下均成立
- 为后续导出 `svg`、`png`、`ico` 以及 Windows 集成提供明确规范

## 设计结论

最终采用 `M-Frame` 方向。

该方向由一个圆角方形外框和一个几何化 `M` 内核组成。它同时表达以下三层含义：

- `Mica Term` 的首字母 `M`
- 一个窗口或工作区容器，呼应桌面应用和 `Mica Alt`
- 两个并排终端面板与一条连接脊，隐含 SSH / SFTP / workspace 的工具属性

这一路径保留了现代终端品牌常见的“几何、克制、技术化”优点，但避免直接落入泛化的 `>_` prompt 图标，也不与现有 SSH 客户端图标形成明显撞脸。

## 备选方向与取舍

### 1. M-Frame

- 优点:
  - 与 `Mica Term` 名称直接关联
  - 适合 Fluent / Mica Alt 的面板化结构
  - 在任务栏小尺寸下仍有清晰主轮廓
  - 品牌独特性强于通用终端 prompt
- 风险:
  - 若内部几何过度复杂，小尺寸会变脏

### 2. Terminal Prompt

- 优点:
  - 一眼可识别为终端产品
  - 语义直接
- 风险:
  - 行业同质化严重
  - 缺乏品牌独占性

### 3. Link Node

- 优点:
  - 更强调连接与远程工作台属性
  - 技术感强
- 风险:
  - 终端属性较弱
  - 品牌首字母关联度不足

最终选择 `M-Frame`。

## Logo 结构

### 外框

- 使用圆角方形作为外轮廓
- 外框代表窗口、工作区与应用容器
- 轮廓不做厚重实体块，更接近一块带层次的系统面板
- 圆角比例遵循 Windows 11 风格，避免尖锐或过度科幻

### 内核

- 内部是几何化 `M`
- 仅保留 3 个主要折点
- 通过两条上升斜线和中间连接脊构成
- 不额外叠加 `>_`、cursor、括号等二次语义装饰
- 目标是让 `M` 成为唯一识别核心

### 轮廓约束

- 在 16px 至 32px 尺寸下依然能一眼辨认
- 优先保证主形，而非细节装饰
- 不依赖描边、玻璃、阴影来提供识别

## 配色方向

采用“中性外壳 + 电蓝内核”的配色结构。

### 深色主版

- 外框底面: 深石墨灰，呼应终端和窗口壳体
- 内部 `M`: `electric-blue`
- 可允许极轻微的同色系亮蓝至冰蓝过渡
- 不使用紫蓝混搭，不做高饱和彩色渐变

### 浅色兼容版

- 保持相同结构，不做整体反白颠覆
- 外框底面提升为云灰或冷浅灰
- `M` 保持同一品牌蓝
- 维持品牌连续性，而不是重新做一套浅色品牌

## 任务栏图标规则

任务栏图标不是全新设计，而是 `app-icon` 的小尺寸特化版。

### 任务栏版保留

- 圆角方形外轮廓
- 中央几何 `M`
- 明确的前后层级对比

### 任务栏版移除

- Mica 质感
- 半透明玻璃感
- 细描边
- 轻阴影
- 所有无法在 24px 以下稳定识别的细节

### 缩放规则

- `M` 线条比品牌主版加粗约 12% 至 18%
- 小于 24px 时不保留次级切角或高光折线
- `16px`、`20px`、`24px` 尺寸允许手工微调，不强制与 256px 完全几何同构

## 资产拆分

最终交付拆成 4 组资产：

### brand-logo

- 用于 README、关于页、启动页
- 可保留较完整的层次与面板感

### app-icon

- 用于安装包、桌面快捷方式、开始菜单
- 是标准品牌主图标

### taskbar-icon

- 用于 Windows 任务栏和窗口角标
- 是小尺寸优化版本

### monochrome-mark

- 用于通知、极小尺寸和单色场景
- 仅保留单色 `M`

## 文件格式与尺寸

- 源文件基准格式: `svg`
- 导出格式: `png` 与 `ico`
- 推荐尺寸:
  - `16`
  - `20`
  - `24`
  - `32`
  - `40`
  - `48`
  - `64`
  - `128`
  - `256`
- `ico` 不能仅依赖 256px 自动缩放，应优先照顾 `16/20/24/32`

## 禁止项

- 不引入第二强调色
- 不使用紫蓝渐变
- 不做荧光描边或泛光发光边
- 不靠 prompt 字符堆砌终端语义
- 不让终端主工作区承担装饰性品牌表达

## 与现有项目风格的关系

这套图标系统需要与当前仓库已确定的风格基线保持一致：

- 对齐 `Command Console Fluent`
- 对齐 `Mica Alt` 分层和 Windows 11 圆角语义
- 对齐 `electric-blue` 单强调色策略
- 对齐深色优先、浅色兼容的主题规范
- 保持终端区中性，不将品牌色扩张进主终端背景

## 后续实现建议

实现阶段应补齐以下工作：

- 新建图标资源目录并提交 `svg` 源文件
- 通过脚本或命令生成多尺寸 `png`
- 生成 Windows 可用的 `ico`
- 将 Windows 应用图标接入构建链
- 将图标资产纳入 `build-win-x64.sh` 的打包产物
- 视需要在 README 中补充图标资源说明
